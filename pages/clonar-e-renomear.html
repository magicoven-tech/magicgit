Clonar e Renomear Repositório

1. Clonar o repositório original

Primeiro, clone o repositório original:

git clone https://github.com/usuario-original/repositorio-original.git

Navegue até a pasta do repositório:

cd repositorio-original

2. Remover o repositório remoto original

Verifique o repositório remoto atual:

git remote -v

Remova a origem remota:

git remote remove origin

Nota: Isso remove apenas a referência remota do seu repositório local. O repositório original no GitHub permanece inalterado.

3. Limpar o histórico do Git (opcional)

Se você quiser começar um novo histórico limpo:

Atenção: Isso irá remover TODO o histórico de commits. Use apenas se tiver certeza!

rm -rf .git && git init

Após executar este comando, você precisará adicionar seus arquivos novamente e fazer o commit inicial:

git add . && git commit -m "Commit inicial"

4. Criar um novo repositório no GitHub

  1. Acesse GitHub e faça login
  2. Clique no botão "+" no canto superior direito e selecione "New repository"
  3. Escolha um nome para o novo repositório
  4. Selecione se será público ou privado
  5. Não inicialize o repositório com README, .gitignore ou licença
  6. Clique em "Create repository"

5. Adicionar o novo repositório remoto

Adicione o novo repositório como origem remota:

git remote add origin https://github.com/seu-usuario/novo-repositorio.git

Dica: Substitua a URL pela URL do seu novo repositório no GitHub.

6. Renomear o branch principal (opcional)

Se desejar renomear o branch principal (por exemplo, de 'main' para 'master' ou vice-versa):

git branch -M main

7. Enviar o código para o novo repositório

Envie o código para o branch principal do novo repositório:

git push -u origin main

Nota: Se estiver usando um branch principal diferente de 'main', substitua 'main' pelo nome do seu branch.

8. Verificar se tudo está correto

Verifique os repositórios remotos configurados:

git remote -v

Verifique o status do seu repositório:

git status

Importante: Se o repositório original for público, lembre-se de que o histórico de commits ainda conterá referências ao repositório original. Para remover completamente o histórico e começar do zero, você precisará criar um novo repositório sem histórico de commits.
