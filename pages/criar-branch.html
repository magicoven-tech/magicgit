Criar e Gerenciar Branches

1. Criar uma nova branch

Para criar uma nova branch a partir da branch atual:

git checkout -b nome-da-nova-branch

Ou, se preferir usar o comando mais moderno (Git 2.23+):

git switch -c nome-da-nova-branch

2. Listar branches existentes

Para ver todas as branches locais:

git branch

Para incluir branches remotas:

git branch -a

3. Trocar entre branches

Para mudar para uma branch existente:

git checkout nome-da-branch

Ou com o comando mais moderno (Git 2.23+):

git switch nome-da-branch

4. Criar branch a partir de um commit específico

git checkout -b nome-da-branch hash-do-commit

Substitua hash-do-commit pelos primeiros caracteres do hash do commit desejado.

5. Enviar uma branch para o repositório remoto

Após criar uma branch local, para enviá-la ao repositório remoto:

git push -u origin nome-da-branch

Dica: O parâmetro -u configura o rastreamento upstream, permitindo que você use apenas git push e git pull posteriormente.

6. Atualizar sua branch local com a branch remota

Se a branch remota foi atualizada por outra pessoa:

git pull origin nome-da-branch

Ou, se você já configurou o rastreamento upstream:

git pull

7. Renomear uma branch

Para renomear a branch atual:

git branch -m novo-nome

Para renomear uma branch diferente da atual:

git branch -m nome-antigo novo-nome

Importante: Se a branch já estiver no repositório remoto, você precisará excluir a branch antiga e enviar a nova.
