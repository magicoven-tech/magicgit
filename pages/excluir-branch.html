Excluir Branches no Git

Atenção: A exclusão de branches é uma operação destrutiva. Certifique-se de que não há alterações não salvas e que a branch não contém commits não mesclados que você deseja manter.

1. Excluir uma branch local

Para excluir uma branch local que já foi mesclada:

git branch -d nome-da-branch

Para forçar a exclusão, mesmo que a branch não tenha sido mesclada (use com cautela!):

git branch -D nome-da-branch

Dica: Você não pode excluir a branch em que está no momento. Mude para outra branch primeiro.

2. Excluir uma branch remota

Para excluir uma branch remota:

git push origin --delete nome-da-branch

Ou usando a sintaxe alternativa:

git push origin :nome-da-branch

Nota: Isso só exclui a branch remota. A branch local ainda existirá em sua máquina.

3. Limpar branches locais que não existem mais no remoto

Para remover automaticamente as branches locais que foram excluídas no repositório remoto:

git fetch --prune

Ou, para fazer isso sempre que você fizer fetch:

git config --global fetch.prune true

4. Excluir múltiplas branches de uma vez

Para excluir várias branches locais que já foram mescladas (exceto main, master e develop):

git branch --merged | grep -v -E "(^\*|main|master|develop)" | xargs git branch -d

Para forçar a exclusão (cuidado!):

git branch --merged | grep -v -E "(^\*|main|master|develop)" | xargs git branch -D

5. Verificar branches mescladas

Antes de excluir, verifique quais branches já foram mescladas na branch atual:

git branch --merged

Para ver branches que NÃO foram mescladas:

git branch --no-merged

6. Recuperar uma branch excluída

Se você excluiu uma branch por engano, ainda pode recuperá-la se lembrar do hash do último commit:

git checkout -b nome-da-branch hash-do-commit

Para encontrar o hash do commit, use o reflog:

git reflog

Procure pelo commit onde a branch foi excluída e use o hash do commit anterior.
