Iniciar um Novo Projeto no Git

1. Inicializar um repositório Git local

Navegue até a pasta do seu projeto e execute:

git init

Isso cria um novo repositório Git vazio na pasta atual.

2. Adicionar arquivos ao staging area

Para adicionar todos os arquivos (exceto os listados no .gitignore):

git add .

Ou para adicionar arquivos específicos:

git add nome-do-arquivo.ext

3. Fazer o primeiro commit

git commit -m "Primeiro commit"

Substitua a mensagem entre aspas por uma descrição significativa das alterações.

4. Criar um repositório no GitHub

  1. Acesse GitHub e faça login
  2. Clique no botão "+" no canto superior direito e selecione "New repository"
  3. Digite um nome para o repositório
  4. Escolha se será público ou privado
  5. Não inicialize com README, .gitignore ou licença (já temos um repositório local)
  6. Clique em "Create repository"

5. Conectar o repositório local ao GitHub

Após criar o repositório no GitHub, você verá instruções. Use o comando fornecido, que será algo como:

git remote add origin https://github.com/seu-usuario/nome-do-repositorio.git

Nota: Substitua a URL pela URL do seu repositório no GitHub.

6. Enviar as alterações para o GitHub

Para enviar seu código para o branch principal (normalmente main ou master):

git push -u origin main

Nota: Se estiver usando um branch principal chamado "master" (versões mais antigas do Git), substitua "main" por "master".

7. Verificação final

Atualize a página do seu repositório no GitHub. Você deve ver todos os seus arquivos lá!

Para verificar o status do seu repositório local a qualquer momento, use:

git status
